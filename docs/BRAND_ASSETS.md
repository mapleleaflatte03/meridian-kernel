# Brand Assets

This repo keeps a minimal Meridian brand asset set under [`docs/assets/`](./assets/)
for repository social preview, release materials, and dark-background documentation.

## Included Assets

- `logo_favicon_64.png` — small icon slots and favicon-scale usage
- `logo_avatar_192.png` — small brand-mark use on dark backgrounds
- `logo_banner_1200x630.jpg` — social preview / share image

## Why The README Does Not Embed These Images

The current Meridian mark is a raster logo with a baked-in dark background and
3D/textured treatment. It works acceptably on dark surfaces, but it is not a
good inline README image for GitHub's default white background.

Until a clean SVG or transparent-background mark exists, the OSS README stays
typography-first. That is an intentional choice, not an omission.

## Usage Guardrails

- Use these assets on dark backgrounds only.
- Do not stretch, recolor, or crop asymmetrically.
- Do not use the banner image inline as hero art in technical docs.
- Prefer the avatar mark for small-header contexts and the banner for social cards.

## Future Improvement

The next brand step should be a flat SVG mark and wordmark lockup that works on
both dark and light backgrounds.
